# Agentic IDE Setup — Cross-Platform Research Synthesis

> **Synthesized from**: Gemini 3.1 Pro (landscape survey), GPT-5.4 (engineering patterns), Claude Opus 4.6 (DX & conventions)
> **Date**: 2026-03-14 | **Target**: MEU-165 `mcp-ide-config` expansion → `zorivest_setup_workspace`

---

## 1. Executive Summary

Three independent research reports converge on a clear architecture for Zorivest's MCP workspace scaffolding tool. The ecosystem is consolidating around **AGENTS.md** as the cross-IDE standard (60K+ repos, 25+ tools, Linux Foundation AAIF stewardship), with a **hub-and-spoke model** where AGENTS.md serves as the vendor-neutral baseline and tool-specific files (CLAUDE.md, `.cursor/rules/`, etc.) add proprietary features. The `.agent/` directory taxonomy is the emerging successor to monolithic root-level config files.

All three reports agree on: static templates shipped via npm, backup+replace idempotency, path confinement security, `.meta.json` version tracking, and placement in the `core` toolset. They diverge on scaffolding scope (progressive tiers vs. full generation) and merge strategy (backup-only vs. AST-aware markdown merging).

The security landscape is more hostile than anticipated: CVE-2025-54136 (MCPoison), 76 malicious agent skills (ToxicSkills audit), 24+ IDE CVEs (IDEsaster disclosure), and privileged injection surfaces in CLAUDE.md make defense-in-depth mandatory.

## 3. Key Insights by Platform

### 🔵 Gemini — Landscape Intelligence (unique contributions)

- **AST-aware markdown merging** is an unmet gap — nobody does it. Using `remark`/`mdast` to parse markdown structure enables intelligent section-level merging instead of primitive append/overwrite.
- **CVE-2025-54136 (MCPoison)**: Cursor trusts approved `mcp.json` indefinitely; attackers swap config post-approval for RCE. Zorivest must never cache approval state for filesystem-modifying tools.
- **Three-tier permission matrix**: Always/Ask First/Never actions in generated rules — directly maps to Zorivest's existing MCP Guard model.
- **Xyzen** (symlink multiplexer): Single AGENTS.md → symlinks to `.cursorrules`, `.windsurfrules`, `copilot-instructions.md`. Simple but fragile on Windows.
- **Everything-Claude-Code (ECC)**: Most advanced agent harness — scaffolds `.agents/`, hooks/, lifecycle events (PreToolUse/PostToolUse). Worth studying for Zorivest's workflow design.
- **`.well-known/mcp.json` draft RFC** (SEP-1649): Future IDE auto-discovery of MCP servers via HTTP endpoints. Zorivest could expose this from the FastAPI backend.

### 🟢 GPT-5.4 — Engineering Precision (unique contributions)

- **Atomic write pattern**: `writeFile(tmp) → rename(tmp, target)` prevents partial files on crash. Essential for multi-file scaffolding.
- **Safe write function**: Complete TypeScript implementation with path validation, backup, atomic write — ready to adapt.
- **Template rendering approach**: Simple string replacement (`{{client}}`) is sufficient; no need for Handlebars/EJS complexity.
- **Error handling contract**: Return structured `{ success, updatedFiles[], skippedFiles[], errors[] }` for agent reporting.
- **`clientVariants()` function**: Generate ALL IDE files regardless of detected IDE (they're small MD files, future-proofs for multi-IDE users).

### 🔴 Claude — DX & Strategy (unique contributions)

- **Progressive disclosure tiers**: Tier 1 (AGENTS.md only, instant), Tier 2 (IDE-specific, detected), Tier 3 (full `.agent/` directory, requested). This is the strongest DX argument — avoid config explosion on first run.
- **Hub-and-spoke model**: Root files are thin shims importing from `.agent/` content. Prevents content duplication, mirrors Copilot's pattern of reading multiple instruction files simultaneously.
- **Cursor migration lessons**: Monolithic files hit scaling limits; `.cursorrules` spawned 36,900-star ecosystem but became too long for LLMs to consume. Glob-based scoping is essential.
- **"Configuration fatigue" risk**: No more than 5-7 files in default scaffolding. Every file must justify existence.
- **ToxicSkills audit**: 76 malicious payloads in 3,984 agent skills. No code signing, no review, no sandbox. Zorivest templates must be static, signed, and auditable.
- **Dual-audience documentation**: AI readership of docs rose 500% in 2025 (GitBook data). Generated docs must serve both humans and AI agents.
- **`<!-- USER RULES BELOW -->` marker**: Partitioned files where the top section is managed (re-scaffolded) and the bottom section is user-owned (preserved).
- **Ken Thompson's bootstrapping parallel**: The trust loop of AI generating its own instructions maps to compiler self-compilation. Solution: human-reviewed seed config that AI can extend but never replace.

### Progressive Disclosure Implementation

```
Tier 1 (default, no args):
  → AGENTS.md (auto-detected project context from package.json, pyproject.toml)
  → IDE-specific root file(s) as thin shims

Tier 2 (detected IDE features):
  → .cursor/rules/*.mdc if Cursor detected
  → CLAUDE.md with @imports if Claude detected
  → .github/copilot-instructions.md if Copilot detected

Tier 3 (explicit --full flag or user request):
  → .agent/
      ├── context/           # Project context, known issues, recent changes
      ├── workflows/         # Step-by-step workflows (.md)
      ├── roles/             # Role definitions (orchestrator, coder, tester, etc.)
      ├── rules/             # Separated by domain (react.md, fastapi.md, etc.)
      ├── docs/              # Help docs, architecture, domain model
      ├── skills/            # Reusable skill packages with SKILL.md
      └── .scaffold-meta.json  # Version, timestamp, hashes, file manifest
```

### Idempotency Strategy (Hybrid)

1. **First run**: Write all files, record hashes in `.scaffold-meta.json`
2. **Subsequent runs**:
   - If file hash matches stored hash → template unchanged AND user unchanged → **skip**
   - If file hash differs from stored hash → user modified → **skip with warning** ("user-modified, not overwriting")
   - If template version is newer than stored → **backup to `.agent/.backup/` then overwrite**, unless user-modified → **conflict warning**
3. **`--force` flag**: Backup all existing, overwrite everything
4. **`<!-- USER RULES BELOW -->` sections**: Preserved across re-scaffolding

### File Write Safety

```typescript
// Core pattern from GPT-5.4, enhanced with Gemini's canonicalization
async function safeWrite(root: string, relPath: string, content: string): Promise<void> {
  const fullPath = path.resolve(root, relPath);
  const realRoot = await fs.realpath(root);
  const realPath = path.dirname(fullPath) === fullPath
    ? fullPath
    : await fs.realpath(path.dirname(fullPath)).then(d => path.join(d, path.basename(fullPath)));

  if (!realPath.startsWith(realRoot)) {
    throw new Error(`Path traversal detected: ${relPath}`);
  }

  await fs.mkdir(path.dirname(fullPath), { recursive: true });

  // Backup existing
  try {
    await fs.access(fullPath);
    const backupDir = path.join(root, '.agent', '.backup');
    await fs.mkdir(backupDir, { recursive: true });
    await fs.copyFile(fullPath, path.join(backupDir, path.basename(fullPath)));
  } catch { /* file doesn't exist */ }

  // Atomic write
  const tmp = fullPath + '.tmp';
  await fs.writeFile(tmp, content, { mode: 0o644 });
  await fs.rename(tmp, fullPath);
}
```

### Provenance Header (all generated files)

```markdown
<!-- Generated by Zorivest MCP v{version} on {date}. Template hash: {sha256}. DO NOT EDIT above this line. -->
```

### Security Defense-in-Depth

| Layer | Implementation |
|-------|---------------|
| **Path confinement** | `path.resolve` + `realpath` + prefix check against project root |
| **Symlink protection** | Resolve symlinks before path validation |
| **Template integrity** | Static assets in npm package, SHA-256 hashes in `.scaffold-meta.json` |
| **Confirmation** | Use MCP confirmation middleware (`get_confirmation_token`) before any writes |
| **Tool annotations** | `readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true` |
| **Content safety** | No executable commands in generated templates, no dynamic content in security sections |
| **Permission matrix** | Generated rules include Always/Ask First/Never action boundaries |
| **MCP Guard** | Respects existing circuit breaker and rate limiting |

## 5. Content Template Strategy

### AGENTS.md (~80-120 lines, Zorivest-specific)

1. **Project identity** — What Zorivest is, tech stack, target users
2. **Architecture** — Frontend/backend boundary, MCP server role, data flow
3. **Build & test commands** — Exact commands (highest-value content per GitHub analysis)
4. **MCP tools reference** — Available toolsets, how to discover/enable them
5. **Domain vocabulary** — Trading terms with precise definitions (portfolio, position, order, P&L)
6. **Security boundaries** — Never actions (execute trades, expose keys, modify prod configs)
7. **File structure** — Key directories and purposes

### IDE-Specific Thin Shims

- **GEMINI.md**: References AGENTS.md + mode mapping (PLANNING/EXECUTION/VERIFICATION)
- **CLAUDE.md**: References AGENTS.md via `@imports` + auto-memory settings
- **CURSOR.md**: References AGENTS.md + activates `.cursor/rules/` if present
- **CODEX.md**: References AGENTS.md + validation-first workflow hints

### `.agent/` Directory Content

- **docs/**: Zorivest MCP tool help — getting started, tool catalog, common workflows, troubleshooting
- **workflows/**: Trade entry workflow, analysis workflow, reporting workflow
- **roles/**: Adapted for trading context (trade analyst, portfolio reviewer, compliance checker)
- **rules/**: Separated by domain — `react-rules.md`, `fastapi-rules.md`, `mcp-rules.md`
- **skills/**: Trade analysis skill, portfolio review skill

| Topic | Tension | Resolution |

| **IDE file generation** | Gemini: symlinks only → GPT: generate all → Claude: thin shims | **Generate all as thin shims** (symlinks fragile on Windows, full copies waste space) |

| **Content length** | Gemini: extensive docs → Claude: AGENTS.md ≤120 lines | **80-120 lines for root files** (Claude's CLAUDE.md ≤200 line guideline applies) |
