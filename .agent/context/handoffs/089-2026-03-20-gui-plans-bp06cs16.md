# MEU-48: GUI Plans (Base + Expansion)

## Scope

Full implementation of the Planning GUI per `06c-gui-planning.md`: `TradePlanPage.tsx` (list+detail split, conviction indicators, form, R:R), `WatchlistPage.tsx` (list+detail split, item table), `PositionCalculatorModal.tsx` (equity-only MVP), and `PlanningLayout.tsx` (tabbed layout). Includes all Tier-1–3 expansion items from MEU-70 scoping (calculator event wiring, ticker autocomplete, market data quote fill, copy-to-clipboard, strategy combobox). Also includes UX polish from MEU-70b (segmented status buttons, conditional trade picker, picker feedback). E2E Wave 4 (position-size.test.ts) gates this MEU.

Build plan reference: [06c-gui-planning.md](../../../../docs/build-plan/06c-gui-planning.md) §16

## Feature Intent Contract

### Intent Statement
Traders can create, update, and evaluate trade plans with full conviction indicators and R:R computation, manage watchlists, size positions using the equity calculator, and link executed plans to specific trades — all from the Planning tab.

### Acceptance Criteria
- AC-1 (Source: 06c-gui-planning.md §16.1): List+detail split for plans: left panel shows conviction emoji, ticker, direction, status badge; detail panel shows full form.
- AC-2 (Source: 06c-gui-planning.md §16.2): Conviction indicators: 🟢 High, 🟡 Medium, 🔴 Low.
- AC-3 (Source: 06c-gui-planning.md §16.3): Live R:R and risk/share computation from entry/stop/target prices.
- AC-4 (Source: 06c-gui-planning.md §16.4): Status transitions via `PATCH /{plan_id}/status` (segmented button group: Draft/Active/Executed/Cancelled).
- AC-5 (Source: 06c-gui-planning.md §16.5): Trade picker — shown always, enabled only when status=Executed, linked to `linked_trade_id` via trade search.
- AC-6 (Source: 06c-gui-planning.md §16.6): Client-side filter by Status + Conviction (API only supports limit/offset).
- AC-7 (Source: 06c-gui-planning.md §16.7): Position calculator — account size, risk %, entry, stop → shares and dollar risk. Global keyboard shortcut `Ctrl+Shift+C`.
- AC-8 (Source: 06c-gui-planning.md §16.8): Watchlist list+detail split; inline add/delete of tickers; price/change columns populated when market data available.
- AC-9 (Source: Local Canon — G5): `refetchInterval: 5_000` on all queries; `data-testid` on all interactive elements.
- AC-10 (Source: E2E SKILL.md Wave 4): `position-size.test.ts` — 2 E2E tests pass: computation correctness + axe accessibility scan.
- AC-11 (Source: emerging-standards.md UX1/UX2/UX3): Segmented status buttons (UX1), responsive enabling with tooltip (UX2), combobox label-in-input pattern (UX3).
- AC-12 (Source: 06c-gui-planning.md §16.9): Calculator pre-fill from trade plan via `zorivest:open-calculator` custom event (entry/stop/target/ticker).

### Negative Cases
- Must NOT: allow status changes when the backend PATCH fails (optimistic rollback via `queryClient.invalidateQueries`)
- Must NOT: show trade picker enabled when status ≠ "executed"
- Must NOT: have WCAG 2.1 AA violations in the calculator modal (enforced by axe E2E scan)

## Design Decisions & Known Risks

- **Decision**: Segmented button group for status instead of `<select>` — **Reasoning**: ≤5 mutually exclusive options; UX1 standard (emerging-standards.md §UX1). **ADR**: inline.
- **Decision**: axe-core injected via `page.evaluate(new Function(src)())` in E2E — **Reasoning**: `@axe-core/playwright` v4.11.1 uses `Target.createTarget` (unsupported in Electron); `page.addScriptTag` blocked by CSP `script-src 'self'`. **ADR**: inline.
- **Decision**: Plan filter dropdowns use `aria-label` (not `<label htmlFor>`) — **Reasoning**: Visual label already present as the first dropdown option ("All Statuses"); `aria-label` satisfies axe `select-name` rule without layout changes. **ADR**: inline.
- **Source Basis**: `docs/build-plan/06c-gui-planning.md §16`; `.agent/docs/emerging-standards.md`; `.agent/skills/e2e-testing/SKILL.md`
- **Risk**: Calculator ticker→quote fill depends on MEU-65 (market data providers). Graceful failure path exits without clearing entry price (AC-C3-4).
