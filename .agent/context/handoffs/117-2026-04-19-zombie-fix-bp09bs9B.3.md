# Handoff: MEU-PW5 — Zombie Run Elimination

> **Status:** ✅ Complete
> **Build Plan:** `docs/build-plan/09b-pipeline-hardening.md` §9B.3
> **Matrix Position:** 49.8

## Summary

Eliminated dual-write persistence bug where both `SchedulingService` and `PipelineRunner` created/finalized run records, causing zombie runs (stuck in "running" after crash). Now `SchedulingService` creates the record with `pending` status and passes `run_id` to `PipelineRunner`, which owns the full lifecycle (pending→running→success/failed).

## Architecture Decision

**Persistence ownership model:**
- `SchedulingService` → creates record (status=`pending`), passes `run_id`
- `PipelineRunner` → owns lifecycle (pending→running→success/failed)
- No post-runner finalization in scheduler — single writer eliminates zombie risk
