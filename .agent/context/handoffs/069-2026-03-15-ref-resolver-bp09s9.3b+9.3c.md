# MEU-84: RefResolver + ConditionEvaluator

## Scope

Implements two pipeline services: RefResolver (resolves `{"ref": "ctx.step.path"}` in step params) and ConditionEvaluator (evaluates `SkipCondition` against pipeline context with 10 operators).

Build plan reference: [09-scheduling.md §9.3b, §9.3c](file:///p:/zorivest/docs/build-plan/09-scheduling.md)

## Feature Intent Contract

### Intent Statement
RefResolver enables step params to reference outputs from prior steps. ConditionEvaluator enables skip_if conditions to control step execution flow.

### Acceptance Criteria
- AC-1: RefResolver.resolve() returns new dict with `{"ref": "ctx.step.path"}` replaced
- AC-2: Nested dict traversal (dot-separated path)
- AC-3: List index traversal (`ctx.step.items.0`)
- AC-4: Non-ref dicts and lists traversed recursively
- AC-5: Missing step/key raises KeyError
- AC-6: Invalid ref format (no "ctx" prefix) raises ValueError
- AC-7: ConditionEvaluator supports all 10 operators (eq, ne, gt, lt, ge, le, in, not_in, is_null, is_not_null)
- AC-8: Returns True when condition is met (step should be skipped)
- AC-9: Missing field/step resolves to None (graceful, unlike RefResolver)
- AC-10: Unknown operator raises ValueError

### Negative Cases
- Must NOT: Treat dict with "ref" + other keys as a ref (only single-key dicts)
- Must NOT: Crash on missing step in ConditionEvaluator (returns None gracefully)

## Design Decisions & Known Risks

- **Decision**: RefResolver raises KeyError for missing refs; ConditionEvaluator returns None — **Reasoning**: Refs are required data; a missing ref is a config error. Skip conditions should gracefully handle missing context (e.g., step hasn't run yet).
- **Decision**: Single-key `{"ref": ...}` detection — **Reasoning**: Dicts with "ref" plus other keys are regular dicts, not references. This prevents false positives.
