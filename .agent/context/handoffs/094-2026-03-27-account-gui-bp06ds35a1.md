## Bug Fixes Applied

| # | Bug | Root Cause | Fix |
|---|-----|-----------|-----|
| BF-1 | "Add New +" freezes GUI | `selectAccount(null)` was a no-op — no create form existed, `AccountDetailPanel` requires non-null `Account` prop | Added `showCreateForm` state + inline `CreateAccountForm` in `AccountsHome` right pane |
| BF-2 | POST /accounts returns 422 | `useCreateAccount` sent `AccountPayload` without `account_id`; API's `CreateAccountRequest` requires it as mandatory | Added `account_id: crypto.randomUUID()` before spreading payload |

## Design Decisions

1. **TanStack Query** over `useState/useEffect` for server state management
2. **Canvas sparkline** instead of Lightweight Charts (per stop condition — avoids heavy dependency)
3. **React Hook Form + Zod** for form validation in `AccountDetailPanel`
4. **G11 custom events** for wizard lifecycle (same pattern as Position Calculator)
5. **Dedup rule** in wizard: balance only saved if value changed from last snapshot
6. **Client-generated UUID** for `account_id` via `crypto.randomUUID()` — backend requires it, GUI generates it
