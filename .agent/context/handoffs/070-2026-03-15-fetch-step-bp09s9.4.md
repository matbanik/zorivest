# MEU-85: FetchStep

## Scope

Implements FetchStep — the first concrete pipeline step type for fetching market data from configured providers. Includes CriteriaResolver (relative/incremental/db_query date resolution), PipelineRateLimiter (per-provider token buckets + global concurrency semaphore), HTTP cache revalidation (ETag/If-Modified-Since), FetchResult provenance envelope with SHA-256 content hash, and freshness TTL per data type with market-hours awareness.

Build plan reference: [09-scheduling.md §9.4](file:///p:/zorivest/docs/build-plan/09-scheduling.md)

## Feature Intent Contract

### Intent Statement
The fetch stage that retrieves market data from external providers with rate limiting, caching, incremental high-water marks, and provenance tracking.

### Acceptance Criteria
- AC-F1: FetchStep auto-registers in STEP_REGISTRY with type_name="fetch"
- AC-F2: Params model validates required fields (provider, data_type) and rejects missing
- AC-F3: FetchResult computes SHA-256 content hash on init
- AC-F4: CriteriaResolver resolves relative date expressions ("-30d" → exact 30-day delta)
- AC-F5: CriteriaResolver resolves incremental criteria using PipelineStateRepository high-water mark
- AC-F6: FetchResult.cache_status defaults to "miss"
- AC-F7: FRESHNESS_TTL has correct positive TTL for all 4 data types
- AC-F8: is_market_closed() returns True on weekends and after 4 PM ET (3 sub-tests)
- AC-F9: PipelineRateLimiter enforces per-provider limiting via execute_with_limits public API
- AC-F10: HTTP cache returns "revalidated" on 304 response with cached data
- AC-F11: FetchStep returns cached content bytes and success status on cache hit
- AC-F12: Params rejects invalid batch_size (>500 or <1)
- AC-F13: UoW exposes pipeline_state attribute as PipelineStateRepository
- AC-F14: PipelineStateRepository.get() returns None for missing + round-trips upsert→get
- AC-F15: Step registration via import verifies type_name and side_effects

### Negative Cases
- Must NOT: Accept batch_size outside [1, 500]
- Must NOT: Return market_closed=False on weekends or after-hours

## Design Decisions & Known Risks

- **Decision**: `_check_cache()` returns None (stub) — **Reasoning**: Real cache integration depends on future infrastructure adapters. The interface is tested via mock patching.
- **Decision**: `PipelineStateModel.last_cursor` used for high-water marks — **Reasoning**: The model uses `last_cursor` (not `last_value`) per schema; tests and CriteriaResolver align to this.
- **Decision**: FetchStep `side_effects=False` — **Reasoning**: Fetch is read-only; transform/store/render have side_effects=True.
- **Decision**: PipelineRateLimiter tested via public `execute_with_limits()` API — **Reasoning**: Avoids coupling tests to private `_limiters` dict.
